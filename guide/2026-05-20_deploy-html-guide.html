一鍵 Deploy HTML
to GitHub Pages

From a single file to a living archive.

01前置條件

不需要服務器、不需要域名、不需要後端。唯一需要的是一個 GitHub 帳號、一個 token、和一份準備好的 HTML 檔案。

需要什麼

  • GitHub 帳號（免費）
  • Personal Access Token（repo 權限）
  • 一個 public repo（例如 Hermes_VPS）
  • 準備好的 HTML 檔案（自包含或使用 CDN 字體/資源）

1.1 為何選 GitHub Pages

其他選項包括 Vercel、Netlify、Cloudflare Pages。但 GitHub Pages 的優勢在於： 它與 Git 版本控制原生整合，每一次 push 都是一次部署； 不需要額外帳號或 CLI 工具； 對於靜態文件（HTML、CSS、JS）來說完全免費、無流量限制。

02七步部署

Step 1 — 準備 Token

前往 GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens。 新增一個 token，選擇 Contents (read+write) 和 Pages (read+write) 權限。 將 token 保存在本地環境變數 GITHUB_TOKEN。

驗證 Token

curl -H "Authorization: token $GITHUB_TOKEN" https://api.github.com/user | grep login

如果回傳你的 GitHub 用戶名，代表 token 有效。

Step 2 — 創建 Repo

可以用 GitHub 網頁手動創建，或用 API 自動化：

API 創建

curl -X POST -H "Authorization: token $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/user/repos \
  -d '{"name":"Hermes_VPS","description":"靜態網站與日誌存檔","private":false}'

Step 3 — Clone 到本地

用 token 組成 URL 進行 clone，避免每次都要輸入密碼：

Clone 指令

git clone "https://${GITHUB_TOKEN}@github.com/用戶名/Hermes_VPS.git"

Step 4 — 放入檔案

將你的 HTML 檔案放入 repo 根目錄。如果想要一個簡單的單頁網站， 把入口檔案命名為 index.html。

Step 5 — Git 提交

設定 git identity 後提交：

提交指令

git config user.email "deploy@hermes.local"

git config user.name "Hermes Agent"

git add .

git commit -m "部署新版本"

Step 6 — Push

git push origin main

Step 7 — 啟用 GitHub Pages

第一次需要手動開啟或用 API 啟用 Pages：

API 啟用 Pages

curl -X POST -H "Authorization: token $GITHUB_TOKEN" \
  https://api.github.com/repos/用戶名/Hermes_VPS/pages \
  -d '{"source":{"branch":"main","path":"/"}}'

等待 5-10 分鐘後，網站會在 https://用戶名.github.io/Hermes_VPS/ 上線。

03遇到的困難

3.1 404 Not Found

第一次部署後最常見的問題。原因通常不是錯誤，而是時間——GitHub Pages 需要 5-10 分鐘來 build 和分發 CDN。 刷新、等待、用無痕視窗測試，大部分情況都是瀏覽器緩存而已。

3.2 Token 權限不足

Fine-grained token 需要明確選擇 repo 的寫入權限。如果用舊版的 Classic token，要確保 repo scope 已勾選。 錯誤訊息通常是 Resource not accessible by personal access token。

3.3 檔案路徑錯誤

GitHub Pages 預設尋找 index.html 作為入口。如果你的主檔叫 my-log.html， 直接訪問根目錄會看到 404。解決方法：重命名為 index.html， 或者用子目錄（例如 log/index.html）。

3.4 Private Repo 的限制

GitHub 免費方案不支援 private repo 的 Pages。如果你的 repo 是 private， 須升級至 GitHub Pro 或改為 public。

04進階用法

4.1 分類目錄

當檔案越來越多時，建議建立子目錄分類：

目錄結構建議

Hermes_VPS/

  ├── log/

  │   ├── index.html          (日誌目錄)

  │   └── 2026-05-20.html

  ├── guide/

  │   └── deploy-guide.html

  ├── artwork/

  │   └── void-mathematics.html

  └── index.html                (主頁)

4.2 自動更新 Table of Content

每次新增檔案後手動更新 TOC 很煩。建議寫一個 Python 腳本， 自動掃描目錄、擷取標題、重新生成 TOC HTML：

自動化要點

  • 遍歷子目錄中的 .html 檔案
  • 用 regex 擷取 <title> 或 <h1> 作為顯示標題
  • 擷取第一段正文作為簡介
  • 重新寫入 table-of-content.html
  • 提交時一併 push 上去

4.3 一次部署多個檔案

不需要每個檔案都獨立 push。把所有檢議完畢的檔案一次過 add、commit、push， GitHub Pages 會一次過 build 整個 site。

05Hermes 實踐方法

5.1 Hermes 如何幫你做

這份指南本身就是由 Hermes Agent 生成並部署的。 模式是：用戶提供結構與內容要求，Hermes 負責寫 HTML、處理 Git 流程、push 上去。 用戶唯一需要的是在 Command Approval 時點頭。

5.2 呼叫過的 Tools

Tool 用途 說明
read_file 讀取現有 HTML 結構 複製舊檔案的排版語法作為新檔模板
write_file 寫入新的 HTML 檔案 在本地生成完整的自包含 HTML
terminal Git clone / commit / push 執行 git 與 curl 指令
execute_code 自動化腳本 遍歷檔案、生成 TOC、擷取標題
session_search 找回舊對話 確認之前用過的指令和配置

5.3 需要權限的命令

以下是部署過程中需要用戶批准的主要命令，含中文解釋。

git clone 與 token

git clone "https://${GITHUB_TOKEN}@github.com/用戶名/Hermes_VPS.git"

目的：將遠端 repo 拉到本地工作目錄。

風陰：低。Token 位於 URL 中，不會泄露到 shell history。

必要性：無法本地編輯遠端 repo 內容，必須先 clone。

curl + python 驗證 API

curl -s -H "Authorization: token $GITHUB_TOKEN" ... | python3 -c "..."

目的：查詢 GitHub API 確認 repo 狀態與 Pages 是否啟用。

風陰：中。管道輸出到 Python 可能有資安風險，但因為是只讀查詢，無法改變任何設定。

必要性：確認 Pages 已啟用才能預測 URL，避免白等。

git rm 與重新結構

git rm 舊檔案 && git mv log/檔案 guide/

目的：重新分類檔案，移除遺留、建立新目錄結構。

風陰：低至中。只影響 Git 歷史，不會刪除檔案本身（可以 git restore 恢復）。

必要性：當需要重組 repo 目錄時使用。

git push --force

git push origin main --force

目的：將本地修改覆蓋遠端 branch。

風陰：中。如果多人協作，force push 會覆蓋他人的提交。

必要性：個人 repo 而且確認沒有其他人同時編輯時安全。

06工作流程清單

以後每次生成新 log 或 guide 時，以下是標準流程。

標準 SOP

  • 1. 寫 HTML 檔案 → 保存至 /opt/data/home/
  • 2. git clone repo 到 /tmp/
  • 3. 複製檔案到對應目錄（log/ / guide/ / artwork/ 之一）
  • 4. 執行 update-toc.py 自動更新該目錄的 TOC
  • 5. git add -A → git commit → git push origin main
  • 6. 等待 1-2 分鐘，檢查 https://用戶名.github.io/Hermes_VPS/

07一句話

最好的部署不是把檔案送上雲端，
而是讓每一次送出去的東西，
都能被未來的自己找到。
